# Golden rules

## 1. The Core Loop Must Be Fun Before Progression

If removing all progression makes the game boring, progression is masking a problem.

Test: Would I still enjoy the first fight?

## 2. Every Run Should Start With a Meaningful Choice

The player should make a decision before entering the first encounter.

Test: Can two runs feel different before combat begins?

## 3. Power Must Require Tradeoffs

The strongest options should come with risks, costs, or constraints.

Test: Is this choice interesting because it is stronger, or because it changes what I sacrifice?

## 4. Builds Should Be Discovered, Not Chosen

The player should gradually uncover what the run is becoming.

Test: Does Floor 1 fully determine the run?

If yes, there's not enough discovery.

## 5. Every Run Should Create a Story

Players should remember interactions, not statistics.

Bad:

I had +40% attack.

Good:

I turned healing into damage and accidentally became immortal.

## 6. Synergies Must Exist at Multiple Scales

Include:

Minor synergies (optimization)
Major synergies (build-defining)
Hidden/scripted synergies (surprises)

Test: Are there both obvious and unexpected combinations?

## 7. Prefer Mechanical Changes Over Numerical Changes

Changing behavior is usually more memorable than changing values.

Bad:

+2 damage

Good:

Damage chains to adjacent enemies

## 8. Runs Must Diverge Meaningfully

A finished run should look and play differently than another finished run.

Test: If I watched two winning runs, could I tell them apart?

## 9. The Game Must Continually Present Interesting Decisions

The player should rarely autopick rewards.

Test: Does this reward make me think?

If not, it's probably filler.

## 10. Repetition Must Produce Discovery

The player repeats the structure.

The player should not repeat the experience.

Test: After 50 runs, am I still occasionally surprised?

For a deckbuilder specifically, I would add one final meta-rule:

## 11. Cards Should Gain Meaning Through Context

A card should not be powerful because of its text alone.

A card becomes powerful because of:

your deck
your relics
your class
your current run

The same card should feel mediocre in one run and incredible in another.

That is where replayability comes from.

If I were evaluating a deckbuilder prototype, these would be the four I would check first:

Is the core loop fun?
Are there meaningful tradeoffs?
Do builds emerge naturally?
Are mechanical synergies more important than numerical scaling?

If those four work, most of the rest can be tuned later.
