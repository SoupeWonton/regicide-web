# The content bar

New content must change a decision, not merely decorate arithmetic.

## Three tests

1. **Decision:** does the player choose differently because this exists?
2. **Visibility:** can the player attribute the outcome to it immediately?
3. **Pressure:** does it remain relevant at the campaign's meaningful difficulty gates?

Content that fails a test should be revised, combined with a stronger rule, or removed.

## Items

- Each item owns one legible rule bend.
- Power comes with a cost, constraint, timing window, or opportunity cost.
- Avoid flat micro-bonuses that duplicate deck or class identity.
- A player should be able to explain why an item changed the run.

## Offers

- Acquisition is itself a decision; do not hide it behind automatic accumulation.
- Present incomparable shapes rather than an obvious best numerical tier.
- The route, current deck, and upcoming pressure should affect the choice.

## Landmarks

- A landmark has one recognizable verb and creates a route tradeoff.
- Do not add a node type when an existing node can carry the decision cleanly.
- Rewards should affect the deck or an existing loop station, not create another wallet.

## Fight rules

- Rules are visible before they can punish the player.
- Prefer mechanical constraints over small percentage adjustments.
- A rule should test a plan without invalidating a class by fiat.

## Measurement

Simulation estimates floors and detects outliers; playtesting judges legibility,
ownership, and fun. Numbers are tuning inputs, not substitutes for these tests.
