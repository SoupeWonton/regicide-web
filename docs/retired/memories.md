# Memories

> ⚠️ **REMOVED from the implementation (solo economy redesign, 2026-06).** Memories
> and the chapter-end memory draft are gone from the code (types, content, engine, UI).
> This document is design history only. Relics — now **two slots per hero** — carry the
> persistent-identity role memories used to hold. See `CLAUDE.md` → "Solo economy redesign."

Memories are hero-lineage progression earned through chapter success.

## Canon Rules

- At chapter end, each surviving hero receives a Memory draft.
- Draft presents 3 choices from a shared global pool.
- v0 target pool size is about 15 Memory designs.
- Memory value baseline is 0.25 CT.
- Memory persists until that hero dies or retires.

## Ownership

Memories are hero-bound, not team-bound.

- Switching to a new campaign lineage does not carry old-team Memories.
- Starting from a later unlocked chapter does not grant assumed Memories.

## Death And Lineage

When a hero dies:
- that hero's Memory is lost with the hero
- replacement lineage flavor can still reference the fallen hero narratively

Replacement linkage is a narrative and bonus-context tool, not automatic Memory transfer of full power.

## Design Intent

Memories should represent earned continuity for surviving heroes and reward long-term team preservation.

They should not invalidate run tension.

## Example Memory Directions

- first kill each encounter grants minor setup bonus
- once per chapter, recover a small hand-quality spike after Hearts trigger
- one role-specific passive that strengthens a class identity without replacing core rules
