# V3.0 integration plan — decided design vs. the live game

How the **decided V3.0 design** (Continent 1 + 2; see
[`../../decisions/2026-06-25-v3-scope-c1-c2.md`](../../decisions/2026-06-25-v3-scope-c1-c2.md))
maps onto the **code that is actually live** on llgames.ca, and the exact path to close the gap.

## Branch reality (verified 2026-06-25)

`Design_V3` and the live branch **`claude/cloudflare-balancing-test-setup-dlanrm`** (the one
deployed to llgames.ca) **diverged** from a common base `c2a371e` — they are not stacked.

- **Live branch = the real game, ahead on gameplay code.** Two feature commits past the base:
  `3090303` (Kingfall rebrand, solo-only lobby, token/overdraw UI, Caravan skip, death rework)
  and `91d3677` (graft-on-kill, 4-core class gate, scripted onboarding tutorial).
- **`Design_V3` = the decided design (docs).** Its only code commit (`8cd57e5`, an early graft
  prototype) is **behind** the live branch's `91d3677`.

**Consequences:**
- A commit on `Design_V3` changes **docs only** — zero gameplay change. llgames.ca deploys from
  the live branch (`render.yaml`), so a `Design_V3` commit is invisible to players.
- **Do not merge `Design_V3`'s code into live** — its graft commit is older; a code merge would
  regress the live game.
- The integration base is the **live branch**, not Design_V3's code.

## Integration path

The live branch is the integration base.

1. **Hygiene** — delete stray 0-byte root files (`+`, `4-slot`, `The`, `and`, `rewards`,
   `superseded.,+,`, two mojibake files); commit `docs/` cleanly on `Design_V3` as the
   design-of-record checkpoint. (`Kingfall/` Obsidian vault and `.codex/` tooling stay untracked.)
2. **Establish the integration branch** — branch from the **live** branch; bring the `docs/` tree
   across (docs paths only, keep live `server/`+`client/`). This — not a Design_V3→live code merge —
   is the correct integration.
3. **Resolve Q0 + collapse flags** — make ascending the single V3 default, strip `provinceMode`,
   retire `tokenFragments`/shop remnants.
4. **Build the V3.0 deltas in dependency order:**
   1. **Graft additive → replacement** semantics (state/UI/tests/saves) — smallest, highest-leverage.
   2. **Class Staff × ladder** rewrite + per-class starting kit — critical path.
   3. **C2 royal gates** — convert 4J/4Q/4K province into the **3/2/1 keep-decision + crown**.
   4. **Forgiveness** — opening ♦/♥ floor + C1→C2 seam reset.
   5. **Tree-unlock meta** on C2 completion (blocked on the 1 / several / all decision).
   - **Deferred to V3.5:** crystals' Full/showdown, equipment/relic roster, Fallen Heroes, C3–C5.

## Open decisions that block the build
- Tree-unlock count (one / several / all) — gates step 4.5.
- Per-class Staff + starting ladder + rung values — gates step 4.2.
- Forgiveness numbers + road/rest cadence — gates step 4.4.
