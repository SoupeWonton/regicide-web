# V3.0 build execution — autonomous slice plan

Operational companion to [`v3.0-integration.md`](v3.0-integration.md) (the *what*). This is
the *how*: slice order, per-slice contracts, and acceptance gates so the entire build runs
**without further design input** until human playtest. Every previously-open product decision
is closed or delegated by [`2026-07-01-v3.0-build-decisions.md`](../../decisions/2026-07-01-v3.0-build-decisions.md);
the live-code ground truth is [`live-code-audit-91d3677.md`](../live-code-audit-91d3677.md).

## Ground rules (apply to every slice)

- **Branch:** `v3-integration`. Never touch `live` / `Design_V3` code.
- **Reference, don't merge:** `experiments/reforge-replenish` (9ecd62f) is a prototype
  reference for grafts; the V3 implementation is built fresh on §F.
- **Acceptance gate per slice:** `cd server && npx tsx scripts/smoke.ts` ends
  "All smoke tests passed ✅"; client typecheck (`vue-tsc --noEmit`) clean; a new smoke
  case covering the slice's core rule; commit per slice with a conventional message.
- **Numbers:** placeholders per plan §A, chosen once and recorded in the slice's contract
  file — never re-litigated mid-build.
- **Contracts:** ability/relic/spell pins land under `docs/delivery/contracts/`
  (`staffs-and-ladders.md`, `relics.md`, `spells.md`) as their slice is built; ambiguous
  readings get a ⚑ flag + the chosen interpretation for playtest review.
- **Deletes come LAST** (slice 9): nothing is removed until its replacement is live.
- **UI:** functional placeholder screens in the existing client style (Vue components
  following `client/src/components/campaign/` patterns); Gab restyles later.
- **Solo only.** Do not touch the quick-game/multiplayer code paths.

## Slice order and content

### 1 · §F card-state model *(foundational)*
Versioned physical card model in `server/campaign/types.ts` + a new `cards.ts`:
stable `physicalId`, `printed {rank,suit}` vs `effective {rank,suit}`, graft provenance
(what replaced what, movable), serialization + schema version bump, projection helpers for
the client (printed vs effective display). Migrate `ownedCards`/`cardTokens` keying off
logical `${suit}${rank}` onto physical IDs with a compatibility shim while old systems remain.

### 2 · Replacement grafts
Route owned-card exact kills → **replace rank OR suit** of one held card (player choice,
`graft_select` phase reused); royal grafts cap value at 10; grafts fire in C1 too (ownership
is the trigger). Additive `hone`/`graft` retired from the offer path (code deleted slice 9).
**Decision 3: no fragment on the graft trigger.**

### 3 · C2 royal gates — 3/2/1 pyramid
Convert the full-court (per `C1_GATE_RANKS`-style stacks): P1 Jack Gate fight → keep 3 of 4;
P2 Queen Gate → keep 2 of 4; P3 King Gate → keep 1 of 4 (crown; betrayer-King abdicates).
Kept royals shuffle in as real deck cards; royals left behind are fought, not recruited.
Royal exact kills graft (cap 10). Victory = King Gate cleared → run complete.

### 4 · Classes — path + Staff
Starting deck: **identical A–5 × 4 suits for all classes** (Decision 1). Class select →
pick 1 of that class's 4 Staffs (the passive signatures in
[`facet-and-linear-candidates`](../../wiki/v3/classes/facet-and-linear-candidates.md));
`siegeText` retired. Entering C2 grants the home-path C2 rung (single ability; simple reveal
animation placeholder); C3/C4/C5 rungs visible-but-locked. C2 clear → all paths unlock (meta).
**Pin all 32 ability contracts** → `contracts/staffs-and-ladders.md` (Decision 9); only C2
rungs + Staffs need working code now (16 + 4 home rungs; non-home rungs = data + locked UI).

### 5 · Forgiveness
Opening hand always contains ≥1 Diamond. **Camp = four-part bundle** (Decision 4): reshuffle
discard → Tavern · **draw up to 5** (top-up, Decision 6) · first attack doubles · **start next
fight with 10 block**. **Seam reset** (automatic, at every province boundary, in the recap):
reshuffle + draw up to 5 — no block, no double.

### 6 · Spells — gauntlet + bracelet
Collapse the 14-spell roster → 4 suit crystals × {Fragment, Half} (Keen Edge ♣ / Quick
Muster ♦ / Guard Up ♠ / Refit ♥ identities carry; Half tier per the §6 table). Fragments:
agnostic, 50/50 post-encounter roll, **uncapped bracelet pool** (Decision 8). Bracelet =
between-encounter placement (equip or sandbag); **Forge landmark** = tier-up (fragments →
Half; Full = V3.5, progress banks silently). **Cast = consume to empty** (Decision 2); cap
one cast per suit per combat; castable over matching-suit immunity. Placeholder numbers:
fragments-per-Half = 2; Rally draw cap = 5. Contracts → `contracts/spells.md`.

### 7 · Equipment — relics
Slot model: `arms|armor|trinket` + `RELIC_SLOTS=2` → **four named slots (Hat/Amulet/Ring/
Cloak), one each**, equipped from a **bag**; **free re-equip at every between-encounter
screen, locked in combat** (Decision 7). Author `relic_v1_design_3.0` (29) from
[`canon/v3/systems/relics.md`](../../canon/v3/systems/relics.md); Hoard + Sainted Scalpel
carry from live; **pin all 29 contracts** → `contracts/relics.md` (Decision 10; Scalpel N=6;
fragment-touching relics use agnostic fragments). Acquisition: Lair (raid) + Caravan
(pay-from-hand discard-total) → bag.

### 8 · Landmarks + provinces
Roster per plan §8: Forge (tier-up), Sanctum (Rearrange — 2 suit-or-value transfers/visit),
Fallen Heroes (start C2-P2; free repeatable Staff swap — one random Staff per class),
Lair, Caravan (pay-from-hand), Camp (slice 5), Shrine (Consecrate — transmute one owned
card's suit or rank, no kill), **Hunt (C1 only — NEW build**: target a missed recruit,
still exact-kill). Rename chapter → province (3 per continent); **C2 provinces mirror the
C1 road skeleton** (Decision 5) with gates as bosses; rest at every seam (slice 5); recap
screen at province/continent boundaries. God-of-Luck wager = animation placeholder.

### 9 · Meta/lineage + cutover *(deletes now)*
Lineage: wipe all on first V3.0 boot (one-line explainer); schema = engineering's call within
the decided shape (C2-clear flag → all paths open; meta banks options, not power). No mid-run
save/resume (single-session run; keep the boundary recap; lineage meta persists). Then the
§11 deletes, each verified against its live replacement: fragment shop, curse system,
additive hone branch, Tower node + handler, `tokenBudget`, `CURATION_CUT`, `provinceMode`
flag + non-ascending branches, spell wallet/shop remnants. Final pass: full smoke + a
scripted end-to-end run (C1 start → King Gate victory) green.

## Definition of done (playtest handoff)

- A fresh `npm run dev` game: class+Staff select → C1 (3 provinces, recruit 6–10, Hunt,
  landmarks) → seam recap → C2 (rung reveal, 3 gates, Fallen Heroes) → King Gate victory
  screen; death = run over. Smoke suite + e2e green; contracts docs complete with ⚑ flags
  collected for Landry's playtest review.
