# Core loop

**Summary:** V3 sequences draw, combine, kill, block, and persist as a shared card-resource problem across turns.

## What it is

The loop is **draw → combine → kill → block → persist**. New mechanics may modify one station but should not add another simultaneous subsystem. (source: [[canon/v3/core-loop|Core loop]])

The stations are not meant to be individually trivial or independent. Their strategic value comes from competition for the same cards across turns, provided the player can reason through the sequence rather than price several unrelated effects at once. (source: [[canon/principles/depth-width-elegance|Depth vs. width]])

Exact kills connect combat to progression: an unowned enemy is recruited, while an owned enemy supplies a permanent rank-or-suit replacement graft. The Ace remains a low-rank starting companion that adds one to a play. (source: [[canon/v3/core-loop|Core loop]])

Each station makes a different claim on the same hand: draw changes future options, combine forms the play, kill decides survival and reward, block reduces payment, and persist determines whether the engine reaches later encounters. Spending an exact-value card on defense can save this turn while forfeiting [[v3/mechanics/exact-kills|recruitment]] next turn; that cross-turn competition is the intended depth. (sources: [[canon/v3/core-loop|Core loop]], [[canon/principles/depth-width-elegance|Depth vs. width]])

The classes reinterpret the chain without adding minigames: [[v3/classes/quartermaster|Quartermaster]] changes Combine, [[v3/classes/executioner|Executioner]] changes Kill, [[v3/classes/sentinel|Sentinel]] changes Block, and [[v3/classes/surgeon|Surgeon]] changes Persist. Draw stays shared, while campaign-only reward phases remain isolated from quick-game Regicide. (sources: [[canon/v3/classes/overview|Classes]], [[canon/v3/constraints|V3 constraints]])

## Confirmed design

- The loop order and five station names are canon.
- New mechanics may modify a station but should not add a simultaneous subsystem.
- Unowned exact kills recruit; owned exact kills create a replacement choice.
- The Ace remains a low-rank companion adding one to a play.
- Campaign mechanics remain isolated from quick-game Regicide. (sources: [[canon/v3/core-loop|Core loop]], [[canon/v3/constraints|V3 constraints]])

## Not yet decided

The complete V3 mutation vocabulary, final class hooks, later-act pressure rules, and some printed-versus-effective card interactions remain open. These questions refine the loop without reopening its accepted sequence. (source: [[proposals/open-design-questions|Active design questions]])
