# Tutorial

**Summary:** The existing scripted tutorial proposal contains useful teaching principles but requires structural revision for current V3.

The proposal favors a short, seeded, learn-by-doing rail with one concept per beat, harmless failure demonstrations, soft highlighting, immediate skip, and no new combat rules. Those are reusable player-experience principles rather than accepted implementation. (source: [[proposals/player-experience/tutorial|Tutorial proposal]])

Likely teaching targets include play and counterattack, suit powers, combinations, exact versus overkill, recruitment versus replacement graft, persistent attrition, and one visible class loophole. The final boundary depends on settled mechanics. (source: [[proposals/open-design-questions|Active design questions]])

The current script is stale: it uses chapter terminology, ends by recruiting a royal, and contains assumptions from the previous campaign structure. Current V3 treats royals as gate bosses rather than ordinary recruits, so the script must not be implemented as written. (sources: [[proposals/player-experience/tutorial|Tutorial proposal]], [[canon/v3/campaign/structure|Campaign structure]])

The original MUST list teaches Hearts, Spades, the Ace companion, exact-kill recruitment, owned-kill replacement, royal immunity, and Jester recovery; attack, Diamonds, Clubs, payment, and combinations are supporting beats. Its design rules remain valuable: one concept per forced-obvious action, one short line, harmless scares, a fixed deck, visible highlighting, immediate skip, and a rewarding finish. (source: [[proposals/player-experience/tutorial|Tutorial proposal]])

Its two-fight script cannot be transferred literally. The Gatekeeper ends in royal recruitment, uses Chapter 1 language, and assumes older royal/Jester behavior. A V3 rewrite should preserve the rail and teaching economy while ending on a valid number-card recruit or replacement and introducing persistent attrition plus one visible class loophole during Claim. (sources: [[proposals/player-experience/tutorial|Tutorial proposal]], [[canon/v3/campaign/structure|Campaign structure]], [[proposals/open-design-questions|Active design questions]])

Guide voice, soft-gate strength, fixed versus selected class, and standalone versus embedded placement remain open. Implementation must remain seeded, use ordinary campaign actions, and avoid a parallel combat mode. (source: [[proposals/player-experience/tutorial|Tutorial proposal]])
