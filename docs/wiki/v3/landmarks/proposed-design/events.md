# Events — *Lore-driven*

**Proposed design (fresh).** One verb: **Lore-driven exception.** See
[[v3/landmarks/proposed-design/philosophy-and-design-orientation|philosophy & orientation]].

## The one decision
> An authored, one-off choice that only exists because the *lore* earns it.

## What it does
Exceptional, authored content — **not a baseline acquisition channel**. Events are written from the
lore later; they are the deliberate exception to the "one verb, one decision" rule, not a recurring
reward type.

## Why route here / opportunity cost
An Event justifies its stop through **story**, not loot. If an Event is really just a disguised
shop/reward, it should be re-filed as the landmark it actually is.

## To design
- [ ] What lore hooks warrant an Event at all?
- [ ] Guardrail: Events must not become a backdoor acquisition channel that dodges the ~3-stop cap.
- [ ] Frequency — rare and memorable, never filler.
