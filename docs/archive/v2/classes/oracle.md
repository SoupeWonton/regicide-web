# Oracle

## Class Identity

Tier: 3 — Weird Hero
Theme: Hidden Information
Guiding question: What do we know that others don't?
Interaction layer: probability, hidden state, and planning leverage

Oracle is information control with consequence. It transforms unknown state into planning advantage, but overuse adds pressure through Omen accumulation.

## Expedition Stability Profile

| Pillar | Rating | Notes |
|---|---|---|
| Shield | 0 | — |
| Access | 1 | top-3 reorder improves draw quality |
| Recovery | 0 | — |
| Initiative | 0 | — |
| Consistency | 3 | information control is the deepest consistency source |

CT per chapter: ~1.0
Specialization bonus: Veil Reader +0.75 / Doom Interpreter +0.5
Note: Oracle's CT comes from converting unknown outcomes into known ones. The Omen cost ensures the system self-regulates.
Primary failure mode addressed: boss transition catastrophe (information is most valuable at chapter thresholds)

Core ability:
- At the start of each encounter, look at top 3 Tavern cards and reorder them.

Cost rule:
- If the party uses this information to force a route commitment change, gain one Omen; at 3 Omen, the next boss gets +1 modifier.

Specializations:
- Veil Reader: once per boss, reveal first two enemies and set their order.
- Doom Interpreter: once per chapter, cancel one boss preparation penalty, but add one permanent scar to the Oracle.

## What Oracle Actually Does

Oracle does not add raw numbers. It reduces uncertainty and lets the party convert camp plans into executable lines.

Oracle is ideal in high-difficulty chapters where information itself is a scarce resource.

## Core Ability Deep Analysis

### Information value

Top-3 reorder is strongest when:
- Quartermaster draw effects are active,
- team needs one specific suit/rank window soon,
- chapter has tight sequence constraints.

Lower value when:
- no draw/recovery line can reach reordered cards in time,
- party changes plan every turn and cannot exploit setup.

### Omen economy

Omen is a strategic pressure valve. It prevents Oracle from becoming free certainty.

Good Omen usage:
- spend Omen to secure chapter-defining route pivot.
- accept one boss modifier only when prep depth can absorb it.

Bad Omen usage:
- frequent route panic pivots for minor gains.

### Interaction map

Strong interactions:
- Quartermaster (reorder + draw equals deterministic access).
- Executioner (threshold planning from known card flow).
- Commander (initiative alignment with known lines).

Risk interactions:
- Burden-heavy Exile plus Omen-heavy Oracle can overstack systemic penalties.

## Specialization Path: Veil Reader

Text:
- Once per boss, reveal first two enemies and set their order.

### What it does strategically

Veil Reader is boss-phase planning compression. It allows the party to front-load the right hand architecture.

Use Veil Reader when:
- boss order uncertainty causes repeated collapses,
- party has enough preparation tools to exploit known order,
- chapter pressure is boss-centric.

### Interactions

Best with:
- Sentinel and Surgeon for preallocated mitigation/recovery windows.
- Hunter Executioner for exact-kill sequencing.

Risk:
- less value in road-heavy chapters where boss order is not the core bottleneck.

Best play pattern:
1. Reveal and order for safest first phase.
2. Assign role priorities before first boss action.
3. Spend one-time prep on predicted hardest transition.

## Specialization Path: Doom Interpreter

Text:
- Once per chapter, cancel one boss preparation penalty. Oracle's top-3 reorder ability is suspended for the remainder of the current road segment.

### What it does strategically

Doom Interpreter converts Oracle's own information access into a team plan rescue. The cost is real: suspending the reorder ability means the party navigates the next stretch of road blind.

Use Doom Interpreter when:
- preparation penalties would break the only viable boss plan,
- Oracle continuity is less critical than chapter clear,
- the suspended road segment is short or nearly finished.

### Interactions

Best with:
- Merchant/Commander prep-heavy plans.
- Scout Follower (partially compensates for lost peek ability).

Risk:
- using it early in a long road segment removes information advantage when it matters most.

Best play pattern:
1. Cancel penalty only when the preparation is plan-critical.
2. Only use late in a road segment when the suspension window is small.
3. Route toward Scout Tower landmarks to restore information access.

## Best Way to Play Oracle

Road:
- reorder with explicit intent (who will draw and when).

Camp:
- choose specialization by identifying true bottleneck: boss order vs prep penalties.

Boss:
- convert information into strict action assignments before first major turn.

## Common Misplays

- reordering without a draw access plan.
- accumulating Omen for low-impact route pivots.
- using Doom Interpreter early in a long road segment.

## Build Guidance

Pick Veil Reader if:
- boss sequencing uncertainty causes most losses.
- team can execute planned order reliably.

Pick Doom Interpreter if:
- prep penalties are the dominant failure driver.
- the road segment where the ability suspends is short.

## Difficulty Impact

Compared to base game:
- lowers uncertainty-driven failure.
- preserves tension by shifting cost into Omen pressure and information suspension.
