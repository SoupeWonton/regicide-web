# Class design workspace

Class design currently has one accepted foundation and two competing progression
models.

## Accepted foundation

[`../../canon/v3/classes/overview.md`](../../canon/v3/classes/overview.md) defines the
four identities and their owned loop stations:

- Sentinel — Block.
- Executioner — Kill.
- Quartermaster — Combine.
- Surgeon — Persist.

No detailed kit or progression model is canon yet.

## Competing models

- [`four-core-classes.md`](four-core-classes.md) — one fixed linear three-tier path per
  class.
- [`facets-and-pressure-permutations.md`](facets-and-pressure-permutations.md) — one
  invariant loophole plus a mutually exclusive facet, multiplied by five-act pressure
  permutations.

## Decision required

Determine whether class depth comes primarily from a fixed linear path or from
horizontal facets that deepen during the expedition. After consensus, update canon, add
a dated decision record, and move only the accepted model into delivery scope.
