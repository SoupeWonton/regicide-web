# Class facets and pressure permutations

> **Status:** Exploratory idea packet, not canon and not a balanced class
> specification.

This packet explores how four deep classes can support broad death/milestone unlocks
without adding permanent statistical power or expanding immediately into a shallow
roster. It also shows how the same class expression can play differently under multiple
five-act pressure configurations.

## Accepted foundations

The following are already canonical:

- V3 begins with four sharply separated class identities.
- Each class owns one existing loop station:
  - Sentinel — Block.
  - Executioner — Kill.
  - Quartermaster — Combine.
  - Surgeon — Persist.
- A class is a starting hand plus one innate loophole.
- The loophole deepens during the expedition without a separate point wallet or relic
  slot tree.
- The campaign is one continuous five-act expedition:
  Claim → Shape → Exploit → Adapt → Master.
- Death and milestones widen future possibilities rather than granting required
  permanent statistical power.

The unresolved question is how class deepening produces both mastery and divergent
runs.

## Vocabulary

### Class

The permanent identity and loop rule the character breaks. Class answers:

> What kind of rule-breaking play is always available to me?

### Facet

One mutually exclusive expression of the class loophole for the current expedition.
A facet changes how the class pursues its verb without adding another class system.

> Sentinel always owns Block; Bastion or Riposte changes what Block is used to achieve.

Only one facet is active in a run. Unlocking more facets expands the candidate pool; it
never stacks them together.

### Run build

The specific combination of recruits, replacement grafts, rare relics, routes, and
encounter adaptations discovered during the expedition. The facet biases interpretation
but does not preselect the completed build.

### Pressure package

A coherent test expressed through the existing card engine, such as attrition, tempo,
adaptation, precision, or resource compression. It asks one dominant question without
adding a parallel economy.

### Pressure permutation

The selection and ordering of pressure packages inside the accepted five-act journey.
Act purposes remain fixed; the concrete tests vary.

## Proposed class model

> **Class establishes the verb. Facet establishes the approach. The conquered deck
> creates the build. The pressure permutation changes what that build must prove.**

One possible progression:

| Act | Class expression |
|---|---|
| 1 — Claim | Learn the base loophole and starting hand. |
| 2 — Shape | Choose/discover one facet after seeing part of the conquered deck. |
| 3 — Exploit | The facet gains its primary payoff. |
| 4 — Adapt | The facet gains a secondary or resilience line. |
| 5 — Master | The facet reaches its capstone under synthesis pressure. |

The facet should not be fully chosen in the lobby. The player needs enough deck context
for the decision to express discovery rather than predetermined construction.

## Example facets

These examples describe identities and progression directions. Names and mechanics are
placeholders.

### Sentinel — Block

| Facet | Identity | Example progression direction |
|---|---|---|
| **Bastion** | Store defensive value across turns. | Exact block → retain surplus → spend stored defense at a later gate. |
| **Riposte** | Turn defensive precision into offense. | Exact block → empower the next attack → create exact-kill windows. |
| **Protector** | Preserve valuable cards while defending. | Block efficiently → protect a chosen card → recover a critical blocker. |

All three modify Block. Bastion plans across time, Riposte converts defense into tempo,
and Protector preserves deck resources.

### Executioner — Kill

| Facet | Identity | Example progression direction |
|---|---|---|
| **Momentum** | One exact kill accelerates the next. | Exact-kill draw → consecutive-kill momentum → chain across a gate sequence. |
| **Bounty** | Exact kills improve conquest opportunities. | Reveal targets → pursue missed recruits → improve duplicate-graft targeting. |
| **Kingslayer** | Prepare a decisive royal execution. | Road kills bank preparation → open a boss window → cascade through royal phases. |

Momentum is immediate tempo, Bounty shapes the conquered deck, and Kingslayer converts
road execution into gate preparation.

### Quartermaster — Combine

| Facet | Identity | Example progression direction |
|---|---|---|
| **Arsenal** | Construct exceptionally large plays. | Raise combination ceiling → widen legal compositions → commit one decisive turn. |
| **Reserve** | Hold the right answer for a later turn. | Expand hand planning → reserve one card → recover it for a gate phase. |
| **Logistics** | Reconfigure how cards combine. | Increase Ace flexibility → manipulate composition → revise a combination before commitment. |

Arsenal maximizes present output, Reserve protects future sequencing, and Logistics
changes composition rules.

### Surgeon — Persist

| Facet | Identity | Example progression direction |
|---|---|---|
| **Triage** | Recover specific cards instead of generic volume. | Choose recovery → target transformed cards → return them closer to play. |
| **Circulation** | Keep Tavern and discard moving continuously. | More recovery windows → discard feeds Tavern → prevent the engine from stalling. |
| **Resurrection** | Recover from near-collapse. | Emergency return → restore a limited package → one late-run system recovery. |

Triage values card quality, Circulation values throughput, and Resurrection values
comeback capacity.

## Example pressure packages

### Attrition

**Question:** Can the persistent deck remain functional when recovery opportunities are
scarce?

Possible tools: longer sequences between rests, Tavern depletion pressure, and gates
with multiple enemies.

### Tempo

**Question:** Can the deck win before visible pressure grows beyond control?

Possible tools: enemy attack escalating across turns, strengthening waves, and a clear
cost for delaying an exact kill.

### Adaptation

**Question:** Can the player use a secondary line when the preferred pattern is
unfavorable?

Possible tools: visible rotating suit immunity, alternating suit pressure, and recruits
that invite flexibility through replacement grafts.

### Precision

**Question:** Can the player preserve exact-kill control while under pressure?

Possible tools: awkward health totals, several acquisition targets, and conflict
between immediate survival and next-turn exact-kill setup.

### Resource compression

**Question:** Can the player operate when fewer useful cards are immediately available?

Possible tools: important cards deeper in the Tavern, consecutive encounters, and
visible limits on draw or recovery timing.

## Example pressure permutations

Acts 1 and 2 retain their accepted Claim and Shape purposes. A permutation primarily
changes how Acts 3–5 realize Exploit, Adapt, and Master.

### The Long War

| Act | Pressure expression |
|---|---|
| 1 | Claim. |
| 2 | Shape. |
| 3 | Attrition. |
| 4 | Resource compression. |
| 5 | Attrition and tempo in telegraphed successive phases. |

### The Rising Court

| Act | Pressure expression |
|---|---|
| 1 | Claim. |
| 2 | Shape. |
| 3 | Tempo. |
| 4 | Precision under escalation. |
| 5 | Successive tempo, precision, and recovery phases. |

### The Shifting Crown

| Act | Pressure expression |
|---|---|
| 1 | Claim. |
| 2 | Shape. |
| 3 | Adaptation. |
| 4 | Rotating suit pressure with awkward exact values. |
| 5 | A known sequence of suit, precision, and attrition tests. |

### The Broken Road

| Act | Pressure expression |
|---|---|
| 1 | Claim with fewer acquisition opportunities. |
| 2 | Shape through directed Hunts. |
| 3 | Resource compression. |
| 4 | Recovery and adaptation. |
| 5 | Mastery with a battered but player-authored deck. |

## How facets and permutations multiply

The same facet should require a different plan under different pressure.

Example: **Sentinel — Riposte**

- In **The Long War**, decide when stored defensive value is worth converting.
- In **The Rising Court**, exact blocks create enough offense to keep pace.
- In **The Shifting Crown**, suit rotation changes which cards can form an exact block.
- In **The Broken Road**, scarce ideal Spades make replacement grafts and secondary
  blocking lines more important.

Example: **Surgeon — Triage**

- Attrition makes targeted recovery the primary engine.
- Tempo forces recovery of offense rather than maximum long-term value.
- Adaptation changes which effective suit or rank must return.
- Precision rewards recovering a numerical answer rather than the strongest card.

## Example meta-unlock sequence

Unlocks introduce new hypotheses rather than permanent bonuses:

- Start: four classes, two facets per class, one default pressure permutation.
- Reach Act 3 with a class: unlock its third facet as a possible future offer.
- Reach Act 4: unlock The Long War.
- Reach a graft milestone: unlock The Shifting Crown.
- Complete an expedition: unlock a faction-specific relic possibility package.
- Complete one faction with all four classes: unlock its mastery permutation.

Offer size remains fixed. If three facets are unlocked, a run can still offer two and
activate one. More unlocked content must not mean more simultaneous passives.

## Breadth without a larger roster

A restrained structure already creates substantial macro variation:

- 4 classes.
- 3 facets each.
- 2 starting-hand packages each.
- 5 pressure permutations.

That is 120 macro configurations before recruits, replacement grafts, relics, routes,
and seeds. The parked V2 classes are therefore optional future expansion, not the
primary breadth plan.

A future class should only be added if it owns an irreducible loop identity the four
cannot express. It should not exist merely to fill an unlock track.

## Guardrails

1. One class owns one permanent verb.
2. One facet is active per expedition.
3. Unlocking facets expands the pool but never stacks class passives.
4. Facet choice occurs after enough deck information exists for an informed decision.
5. A facet modifies its class's existing loop station rather than adding a subsystem.
6. Pressure packages use the existing card engine and remain telegraphed.
7. Act 5 sequences pressures across time instead of resolving unrelated axes at once.
8. Unlocks create new plans, never strict numerical upgrades.
9. Two runs with the same class should be distinguishable without reading the class
   name or portrait.

## Suggested initial content budget

For the first complete test of this model:

- Four classes.
- Two facets per class available initially.
- One additional unlockable facet per class.
- One default pressure permutation.
- Two unlockable pressure permutations.
- A constant offer: choose one facet from two.

This is enough to test the multiplication model without designing twelve finished class
trees and five full campaign variants at once.

## Decisions this packet does not make

- Whether facets replace the existing linear three-tier class proposal.
- Exact facet effects, numbers, names, or unlock conditions.
- Whether facet choice occurs in Act 2 or another milestone.
- Whether pressure permutations are selected, faction-defined, or seeded.
- Whether alternative starting-hand packages belong in the initial scope.
- Whether any parked V2 class returns.

## Next class-design decision

> Does each class use one fixed linear progression, or one invariant loophole plus a
> mutually exclusive facet that deepens through Acts 2–5?

The facet model is the recommended direction to develop because it creates horizontal
meta breadth while preserving four legible class identities and a fixed complexity
budget.
